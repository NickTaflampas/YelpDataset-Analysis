Preproccessing¶

The data we have is decent, but for the purposes of guessing a function, we sort our values using A as our index. Therefore, the new pairs are the same, but the values of A are in order.

In [2]:
A = []
B = []
C = []
file = open("data.csv", "r")
file.readline() #Skip first line

for line in file:
    spl = line.split(",")
    A.append(float(spl[0]))
    B.append(float(spl[1]))
    C.append(float(spl[2]))

#Sorting the A values for Plotting
BFunc = {A[i] : B[i] for i in range(len(A))}
CFunc = {A[i] : C[i] for i in range(len(A))}

sortedB = dict(sorted(BFunc.items()))
sortedC = dict(sorted(CFunc.items()))

A = [i for i in sortedB.keys()]
B = [i for i in sortedB.values()]
C = [i for i in sortedC.values()]

Analysis: A First View¶

After plotting the data, we can immediatly see an Exponential Relationship on both functions. By tweaking the data, we can even estimate a mathematical formula for each function.

  • B(A) = 1.7A
  • C(A) = 1.55A

This of course is just an eye-estimate. And we immediatly see that B function can't be approached with our formula.

Analysis: Log-Log Plot¶

Plotting our data in a log-log plot, shows an incredible find.

  • C function continuous to follow an exponential manner, thus proving that it is actually exponential, as we expected.
  • B function on the other hand, follows a linear model! This means that the original B function follows a polynomial manner!

With this new information, we can change B's formula and attempt one with a polynomial model. By tweaking the generalized formula, we end up with the following functions:

  • B(A) = 1.5A2
  • C(A) = 1.55A
